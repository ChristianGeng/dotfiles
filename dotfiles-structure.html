Dotfiles Structure

This visualization shows the structure of the dotfiles repository and explains the purpose of each file.

Deployment: These files should be copied to a .dotfiles directory in your home folder and then symlinked to their appropriate locations. Use the deploy.sh script to automate this process.

Repository Structure

dotfiles/ Root directory
README.md Instructions for using this repository
README.org Original documentation in Org format
deploy.sh Script to deploy dotfiles to a new machine
dotfiles-structure.html This visualization file
bash/ Bash configuration files
.bashrc Main bash configuration
.bash_aliases Aliases for bash commands
.bash_path PATH environment variable configuration
.shell_scripts Custom shell functions and utilities
.dotfiles/ Other configuration files
.gitconfig Git configuration
.nvmrc Node Version Manager configuration

Deployment Process

The deployment process involves:

  1. Creating a .dotfiles directory in your home folder
  2. Copying all configuration files to this directory
  3. Creating symlinks from your home directory to the files in .dotfiles

You can automate this process by running the deploy.sh script:

chmod +x deploy.sh
./deploy.sh

File Descriptions

.bashrc

The main bash configuration file. It sets up the shell environment, loads other configuration files, and configures various bash behaviors.

.bash_aliases

Contains aliases for commonly used commands to make them shorter or add default options.

.bash_path

Configures the PATH environment variable to include directories with executable files.

.shell_scripts

Contains custom shell functions and utilities for various tasks like file manipulation, system information, and more.

.gitconfig

Configures Git settings like user information, aliases, and behavior.

.nvmrc

Specifies the Node.js version to use with Node Version Manager (NVM).
